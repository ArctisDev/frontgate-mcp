# Skill: Creating Clear Focal Points

## Category
Hierarchy

## Description
Every screen should have a single focal point — the element that draws the eye first and communicates the page's primary purpose. When agents generate layouts where every element has equal visual prominence, or where decorative elements compete with functional ones, the user has no entry point into the page. Without a focal point, users must consciously decide where to start reading, which increases effort and reduces engagement.

This skill teaching creating an unmistakable focal point on every screen. On a dashboard, this might be the primary metric card in a larger size with a bold number. On a settings page, it might be the most important toggle group with a prominent heading. On a landing page, it is the hero headline and primary CTA. Use size, contrast, color, and positioning to make the focal point dominant. Everything else on the page should support or contextualize the focal point rather than compete with it.

## Expected Outcome
- Every screen has a single dominant focal point
- Focal point communicates the page's primary purpose immediately
- Surrounding elements support the focal point rather than competing
- User's eye is drawn to the focal point within the first second

## Heuristic Triggers
- `CountNewTopLevelComponents`: Excessive new components can dilute focal point clarity
- `FindOversizedSidebarClasses`: Oversized sidebars can steal focus from the main content area

## Scoring Axes Most Affected
- ux_clarity: A clear focal point gives users an immediate entry point into the page
- product_alignment: The focal point should embody the page's core user value
